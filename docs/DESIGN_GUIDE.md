# 취향 (Chuihyang) 디자인 가이드

> **이 문서는 취향 앱의 UI/UX 디자인 원칙과 스타일 가이드입니다.**
> 모든 화면과 컴포넌트는 이 문서의 원칙을 따릅니다.

---

## 1. 디자인 컨셉

### 핵심 키워드
- **밝고 따뜻한** — 크림/베이지 톤의 따뜻한 배경
- **동글동글한** — 큰 borderRadius, pill 버튼, 부드러운 모서리
- **친근한** — 딱딱하지 않은 캐주얼한 톤, 이모지 활용
- **깔끔한** — 불필요한 장식 배제, 여백 활용

### 브랜드 아이덴티티
- **앱 이름**: 취향 (Chuihyang)
- **컨셉**: 나만의 담금주를 기록하는 따뜻한 공간
- **모드**: 라이트 모드 전용 (다크 모드 없음)

### 레시피 브랜드 컬러 (변경 불가)
각 담금주 레시피의 고유 컬러는 앱 전체에서 일관되게 사용합니다.

| 레시피 | 색상 코드 | 컬러 |
|--------|----------|------|
| 야레야레 (전통) | `#025830` | 진녹색 |
| 블라블라 (허브) | `#20407c` | 진청색 |
| 오즈 (진한 맛) | `#ab1e4b` | 진분홍색 |
| 파친코 (과일) | `#eaa728` | 주황색 |
| 계애바 (클래식) | `#921e22` | 진빨강색 |

이 컬러들은 프로젝트 카드, 진행률 바, 상세 화면 헤더 등에서 accent로 사용됩니다.

| **xs** | 4px | 아이콘-텍스트 간격 |

| **md** | 12px | 기본 간격 |

| **lg** | 20px | 섹션 간 간격 |
| **xl** | 24px | 화면 좌우 패딩 |
| **2xl** | 32px | 큰 섹션 간격 |
| **3xl** | 48px | 화면 상하 패딩 |

### 화면 패딩
- 좌우 패딩: **24px** (고정)
- 상단 패딩: Safe Area + **16px**
- 하단 패딩: Safe Area + **16px**

## 5. Border Radius

| 이름 | 값 | 용도 |
|------|-----|------|
| **sm** | 8px | 작은 태그, 뱃지 |
| **md** | 12px | 입력 필드, 작은 카드 |
| **lg** | 16px | 일반 카드 |
| **xl** | 20px | 큰 카드, 모달 |
| **2xl** | 24px | 메인 카드, 버튼 |
| **full** | 9999px | 원형 (아바타, pill 버튼) |

### 원칙
- **카드**: 20~24px (동글동글)
- **버튼**: 24px (pill-like)
- **입력 필드**: 12~16px
- **아바타**: full (원형)

## 6. 그림자

단순한 그림자 2종만 사용합니다. (기존 neumorphism/glass 6종 → 2종으로 축소)

### Shadow - Soft
```
shadowColor: '#000'
shadowOffset: { width: 0, height: 2 }
shadowOpacity: 0.06
shadowRadius: 8
elevation: 2
```
용도: 일반 카드, 입력 필드

### Shadow - Medium
```
shadowColor: '#000'
shadowOffset: { width: 0, height: 4 }
shadowOpacity: 0.1
shadowRadius: 16
elevation: 4
```
용도: 떠있는 요소, FAB, 모달

## 9. 애니메이션

### 원칙
- 빠르고 가볍게 (150~300ms)
- 과도한 애니메이션 지양
- 의미 있는 전환만 (화면 전환, 버튼 누름, 토스트)

### 버튼 프레스
- scale: 0.97 (살짝만)
- duration: 100ms
- spring tension: 300

### 화면 전환
- fade + slide (기본 Expo Router 전환)
- 커스텀 전환 최소화

---

## 10. UX 원칙

### 빈 상태
- 항상 일러스트 또는 이모지 + 설명 텍스트 + CTA 버튼 제공
- "아직 담금주가 없어요" 같은 따뜻한 톤

### 피드백
- 액션 후 즉각적인 피드백 (로딩 인디케이터, 성공 토스트)
- 에러는 Alert 대신 인라인 메시지 선호

### 접근성
- 터치 타겟 최소 44x44px
- 충분한 색상 대비 (텍스트 4.5:1 이상)
- 레이블 명확하게
